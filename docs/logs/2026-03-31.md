# Session Log — 2026-03-31

## What was built
- Diagnosed and fixed first-line-clear freeze (iOS Safari compositor stall)
- Pre-created effect element pool at game init (particles, popup, combo text, flash overlay)
- Tiered clear effects: single, double, triple, quad+, cross, board clear
- Outline-only drag preview (dashed border replacing semi-transparent fill)
- Piece-return: drag below board cancels placement
- Removed debug instrumentation

## What failed / dead ends
- Assumed CSS `filter: brightness()` was the cause — removed it, freeze persisted
- Assumed AudioContext was blocking — disabled sound, freeze persisted
- Assumed `will-change: transform, opacity` on 64 cells was the cause — removed, freeze persisted
- Assumed passive touchend event was the cause — fixed, freeze persisted
- Assumed `getBoundingClientRect` layout flush was the cause — removed, freeze persisted
- Binary search isolation finally found it: creating new animated DOM elements during first clear

## Bugs found
- `newGame()` called `effectsContainer.innerHTML = ''` which destroyed pre-created pool elements
- Cancel zone for piece-return (`boardBottom + stride`) was too tight — bottom row unreachable with mobile Y offset. Fixed to `boardBottom + stride * 3`
- touchend listener missing `{ passive: false }` — could delay event delivery on iOS

## Decisions made
- All visual effects use pre-created element pools — no DOM creation during gameplay
- Particles use a rotating pool of 24 elements
- Score popup and combo text are single reusable elements
- Ambient background particles disabled (compositor cost not worth it)
- Clear animation uses transform + opacity only (no CSS filter)
