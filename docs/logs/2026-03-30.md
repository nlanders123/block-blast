# Session Log — 2026-03-30

## What was built
- Floating drag clone system — piece lifts from tray at board cell size, follows finger with Y offset
- Service worker rewrite — cache-first (broken) → network-first with skipWaiting + old cache purge
- clearHighlight fix — no longer strips colour classes from placed blocks (checks for `filled` class)
- Performance: cached cell DOM refs in 2D array, cached layout values per drag session, batched DOM reads/writes in line clear
- Line clear animation shortened from 900ms to 200ms
- Preview cells now pulse at 30-50% opacity to distinguish from placed blocks
- Performance instrumentation overlay (temporary debug tool)

## What failed / dead ends
- `KevinGu/blockblast` repo — SEO landing page, zero game code
- `not1010` clone — mouse-only, Canvas-based, no touch support, minimal drag feedback
- Assumed first-clear delay was: AudioContext init (wrong), CSS animation cold-start (wrong), layout thrashing (partially right but not the main issue), service worker caching (wrong for private tabs)
- Root cause took 8+ deploys to find: drag preview cells visually identical to placed cells, making rows look "complete" before piece is dropped

## Bugs found
- Service worker `block-royale-v1` served stale JS forever (no update logic)
- clearHighlight stripped colour classes from placed blocks when drag preview overlapped them
- getCellElement used querySelector per call (hundreds of times per drag)
- getComputedStyle called on every drag frame (60x/sec layout thrashing)
- Line clear animation was 900ms total (500ms squish + 400ms timeout) — felt frozen
- Preview cells at opacity 0.6 indistinguishable from placed cells during gameplay

## Decisions made
- Network-first service worker (always fetch fresh, cache for offline only)
- Cell DOM refs cached at board creation time
- Layout values cached at drag start, not per frame
- Preview cells need clear visual distinction from placed blocks (pulse + lower opacity)
